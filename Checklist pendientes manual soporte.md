# Checklist pendientes manual de soporte

Lista de tareas pendientes para completar y mejorar el manual de soporte de Circles Learning.

---

## TODOs explícitos en el manual

- [ ] **Completar pasos para subir entrega grupal desde Admin** (sección 3.4) — Actualmente tiene un `<!-- TODO -->`. Documentar el paso a paso para que un asistente suba una entrega en nombre de un círculo.
- [ ] **Completar pasos para subir entrega individual desde Admin** (sección 3.4) — Mismo caso. Documentar cómo subir una entrega por un usuario individual.

## Secciones que necesitan más detalle

- [ ] **Ampliar sección de Sistema de Tickets** (sección 4.1) — Actualmente solo dice que permite gestionar solicitudes de forma organizada, pero no explica cómo crear un ticket, qué campos completar, cómo categorizarlos, ni cómo cambiar estados. Es la sección más incompleta del manual.
- [ ] **Detallar el proceso de responder chats escalados** (sección 4.2) — Dice "selecciona la pestaña Human Support" pero no explica el flujo completo: cómo tomar un chat, cómo responder, qué hacer si necesitas tiempo, etc.
- [ ] **Documentar cómo tomar conversaciones del chatbot** (sección 4.2) — Solo dice "toma la conversación manualmente" sin explicar los pasos concretos.

## Contenido nuevo por agregar

- [ ] **Agregar plantillas de WhatsApp** (sección 7) — Actualmente las plantillas solo cubren plataforma y correo. Faltan mensajes estandarizados específicos para WhatsApp (primer contacto, seguimiento de inactividad, confirmación de baja, etc.).
- [ ] **Agregar plantilla de correo de bienvenida/credenciales** (sección 7.2) — No hay plantilla para el correo inicial con credenciales de acceso.
- [ ] **Agregar plantillas de correo para casos específicos de soporte** (sección 7.2) — Crear plantillas estandarizadas para los casos más frecuentes que manejan los asistentes por email (ej: usuario no puede acceder, problema con entrega, reubicación de círculo, seguimiento de inactividad, etc.). Esto reduce tiempos de respuesta y asegura consistencia.
- [ ] **Agregar sección de métricas y KPIs del equipo** — No existe una referencia a qué indicadores se miden (tasa de retención, tiempo promedio de respuesta, tickets resueltos por semana, etc.) para que el equipo sepa qué se espera.
- [ ] **Documentar el uso de la Base de Datos en más detalle** (sección 5) — Explicar convenciones para el campo "Comentarios Asistente" y "Comentarios Coordinador": qué tipo de notas registrar, formato sugerido, etc.
- [ ] **Agregar sección de errores técnicos conocidos** — Recopilar los bugs o problemas técnicos recurrentes de la plataforma Circles con sus soluciones conocidas o workarounds.
- [ ] **Documentar el proceso de cambio de correo** — Se menciona que se escala a Coordinación, pero no hay un procedimiento detallado de los pasos técnicos involucrados.

## Mejoras generales

- [ ] **Eliminar `Manual-Soporte.md`** — Es una versión incompleta (le falta la subsección "Cómo dar de baja a usuarios"). Quedarse solo con `manual de soporte V2.2.md`.
- [ ] **Generalizar la plantilla de contraseña** (sección 7.2) — La plantilla actual menciona "correo de INACAP" de forma específica. Hacerla genérica o parametrizable para cualquier institución.
- [ ] **Agregar índice con links internos** — Para facilitar la navegación rápida dentro del documento.

## Proceso pendiente de diseñar

- [ ] **Diseñar sistema de gestión del correo compartido de soporte** — Actualmente todos los asistentes usan la misma cuenta de Gmail, lo que causa: (1) correos que un asistente abre y no son para él quedan como "leídos", (2) otros asistentes no los revisan porque aparecen leídos, (3) se pierden consultas de docentes. Necesitamos un sistema que permita asignar correos a asistentes específicos, mantener visibilidad de qué está pendiente, y evitar que se pierdan mensajes. Ver opciones evaluadas en `sistema-correo-compartido.md`.
